Method Overloading Priority Order (Very Important)

Java resolves overloaded methods using this order:

1 Exact Match
2 Widening
3 Autoboxing
4 Varargs

Example priority chain:

int → long   (widening)
int → Integer   (boxing)
int → int...   (varargs)
Real Interview Question Example
void test(int a)
void test(long a)
void test(Integer a)
void test(int... a)

Calling:

test(10);

Output:

test(int a)

Because exact match has highest priority.

---------------------------------------------------------------

Java does not support default values for function (method) parameters like some other languages (e.g., Python, C++, JavaScript).

In Java, every argument must be explicitly passed when calling a method.

Example (Invalid in Java ❌)
    public void greet(String name = "Guest") {   // ❌ Not allowed
        System.out.println("Hello " + name);
    }

Java will give a compile-time error because parameter initialization is not allowed in the method signature.

How Java Handles This (Method Overloading)

Java typically solves this using method overloading.

---------------------------------------------------------------

Why Java Does Not Support Default Parameters

Java designers preferred method overloading instead of default arguments to:

keep method calls explicit

avoid ambiguity in method resolution

maintain backward compatibility

Real-world Example (like Java libraries)

For example in classes like String or Math, Java often provides multiple overloaded methods instead of default parameters.
